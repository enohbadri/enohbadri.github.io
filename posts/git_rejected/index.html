Cara Mengatasi Git Push Rejected di github

Cara Mengatasi Git Push Rejected - Kali ini saya akan membagikan tips ketika sedang git push namun rejected, saya sudah mencari referensi kemana mana namun membuahkan solusi, error rejected yang di maksud anda bisa lihat seperti gambar di bawah ini

Info
Error tersebut terjadi ketika akan melakukan perintah git push origin master dan seketika itu muncul error notifikasi error rejected
$ git push origin master
To https://github.com/acchoblues/CalendarEventManagement.git
 ! [rejected]        master -> master (non-fast-forward)
error: failed to push some refs to 'https://github.com/acchoblues/CalendarEventManagement.git'
hint: Updates were rejected because the tip of your current branch is behind
hint: its remote counterpart. Integrate the remote changes (e.g.
hint: 'git pull ...') before pushing again.
hint: See the 'Note about fast-forwards' in 'git push --help' for details.

setelah pencarian yang panjang akhirnya saya menemukan solusi, ternyata kode git push yang saya gunakan ada yang kurang dibagian akhir saya menggunakan git push origin master berkali - kali pu hasilnya tetap sama rejected, ternyata untuk mengatasi masalah tersebut hanya butuh penambahan saat melakukan git push dengan –force atau -f maka saya coba menggunakan git push origin master –force dan berhasil upload ke github atau anda juga bisa menggunakan kode push yang ini git push origin master -f

Jadi jangan khawatir dan bingung lagi jika anda menemukan error rejected saat git push ke repository anda di github, semoga informasi singkat ini berguna bagi anda yang membacanya, tetap semangat dan jangan pernah menyerah.
