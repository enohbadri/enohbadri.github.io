Cara Mengatasi Git Push Rejected

Cara Mengatasi Git Push Rejected - Kali ini saya akan membagikan tips bagaimana cara mengatasi error git push rejected, error rejected yang di maksud adalah seperti syntax di bawah ini

$ git push origin master
To https://github.com/account/repo.git
 ! [rejected]        master -> master (non-fast-forward)
error: failed to push some refs to 'https://github.com/account/repository.git'
hint: Updates were rejected because the tip of your current branch is behind
hint: its remote counterpart. Integrate the remote changes (e.g.
hint: 'git pull ...') before pushing again.
hint: See the 'Note about fast-forwards' in 'git push --help' for details.
Info
Error tersebut terjadi ketika akan melakukan perintah git push origin master dan seketika itu muncul error notifikasi error rejected

setelah muter-muter di google dan tanya chat-GPT akhirnya saya menemukan solusi, ternyata kode git push yang saya gunakan ada yang kurang dibagian akhir saya menggunakan git push origin master berkali - kali pu hasilnya tetap sama rejected, ternyata untuk mengatasi masalah tersebut hanya butuh penambahan saat melakukan git push dengan –force atau -f maka saya coba menggunakan perintah tersebut.

Cara Mengatasi Git Push Rejected

git push origin master --force

Atau anda juga bisa menggunakan perintah ini :

git push origin master -f

Jadi jangan khawatir ketika menemukan error rejected saat git push ke repository anda di github, semoga informasi singkat ini bermanfaat, tetap semangat. Terimakasih
